### Key Rules

#### âœ… DO:
- **Write conversationally** - imagine you're talking to someone
- **Use short sentences** - easier for listeners to follow
- **Break into paragraphs** - natural pauses in speech
- **Include bullet points** - when listing features or specs
- **Define technical terms** - explain acronyms the first time
- **Use active voice** - "Apply the topcoat" not "The topcoat should be applied"

#### âŒ DON'T:
- Don't use all caps except for product names (ARROYOÂ®)
- Don't write super long paragraphs (3-4 sentences max)
- Don't assume technical knowledge
- Don't use jargon without explanation

## ğŸ“ Script Writing Best Practices

### 1. Conversational Tone
**Bad:** "The aforementioned topcoat product possesses characteristics..."  
**Good:** "This topcoat has several standout features..."

### 2. Clear Structure
Each slide should have:
- **Title** - Clear, descriptive (5-10 words)
- **Narrative** - 100-200 words of spoken content
- **Duration** - Aim for 1-2 minutes per slide

### 3. Product Terminology

#### Always Include Trademarks:
- ARROYOÂ® (with Â®)
- M.L. Campbell (no special marks)
- Any branded product names

#### Technical Terms to Define:
- VOC (Volatile Organic Compounds)
- 1K (one-component)
- Waterborne (water-based)
- Topcoat (protective finishing layer)

### 4. Pacing and Flow

**Ideal slide narration:**
- Opening: What we'll cover (10-15 sec)
- Body: Main content (45-60 sec)
- Closing: Transition or summary (10-15 sec)
- **Total: 60-90 seconds per slide**

### 5. Audience Awareness

Write for **professional finishers** who:
- Understand basic woodworking
- May be new to waterborne products
- Want practical, actionable information
- Value both performance and safety

## ğŸ“ Common Edits and Fixes

### Fixing Typos
Just correct them! Small typos can be fixed anytime.

### Improving Clarity
If something sounds confusing, rewrite it more simply:
- **Before:** "Utilize the applicator to achieve optimal coverage"
- **After:** "Use a brush or sprayer for even coverage"

### Adding Detail
If content feels thin, add:
- Specific examples
- Real-world applications
- Benefits or advantages
- Common questions answered

### Reducing Length
If a slide is too long (>150 words), try:
- Splitting into two slides
- Cutting unnecessary words
- Moving details to another module

### Common Questions

**Q: How do I preview my changes?**  
A: GitHub shows a "Preview" tab when editing - click it to see formatting.

**Q: Can I undo changes?**  
A: Yes! GitHub keeps complete history. Click "History" to see past versions and restore them.

**Q: What if I make a mistake?**  
A: Don't worry! All changes are tracked and reversible. We can restore any previous version.

**Q: How do I suggest a change without making it official?**  
A: Create a "Pull Request" - there's an option when committing that says "Create a new branch and start a pull request."

**Q: Can multiple people edit at once?**  
A: GitHub will warn you if someone else edited the same file. You can merge changes or coordinate timing.
